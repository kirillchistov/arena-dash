# Arena Dash

Arena Dash — браузерная 2D .io-арена с видом сверху.  
Игроки заходят в быстрые матчи, двигаются по небольшой карте, собирают очки и стараются выжить дольше соперников.

## Цели проекта

- Отработать навыки командной разработки (3 разработчика).
- Реализовать простой реальные-тайм мультиплеер через WebSocket.
- Показать базовую авторизацию с ролями **user** и **admin**.
- Собрать полный цикл: от прототипа до деплоя.

## Стек технологий

**Клиент**

- TypeScript
- Vite
- HTML5 Canvas 2D

**Сервер**

- Node.js
- Express (HTTP API)
- ws (WebSocket-сервер)
- JWT (авторизация, позже)

## Архитектура (высокоуровнево)
**Клиент:**
- Рендер арены и сущностей через Canvas.
- Сбор ввода пользователя (клавиатура/мышь).
- Отправка инпута на сервер и отображение состояния игры.

**Сервер:**
- Авторитативная модель игры (позиции, очки, столкновения).
- Управление комнатами и матчами.
- Авторизация пользователей и разграничение ролей.

## План спринтов (драфт)
**Спринт 1**
- Настроен монорепозиторий (`client/` + `server/`).
- Клиент:
  - Vite + TypeScript.
  - Canvas-экран с простой заглушкой арены и «игроком» в центре.
  - WebSocket-клиент, который подключается к серверу и логирует сообщения.
- Сервер:
  - Express-приложение с эндпоинтом `/health`.
  - WebSocket-эхо-сервер на том же порту (3000).

**Спринт 2 (текущий: игровое ядро)**
- Модель комнаты и игроков на сервере.
- Протокол сообщений (join, input, state).
- Простое движение и коллизии.

**Спринт 3 (авторизация и роли)**
- Регистрация/логин, JWT.
- Роли user/admin, защищённые эндпоинты.
- Ограничение доступа к админ-функциям.

**Спринт 4 (полировка и деплой)**
- UI/UX, эффекты, таблица лидеров.
- Тесты, CI, деплой клиента и сервера.

## Команда
- Dev A (Frontend) — Canvas, UI, клиентская сетевая логика.
- Dev B (Backend) — WebSocket-сервер, игровая логика, авторизация.
- Dev C (Инфраструктура) — настройка проекта, тесты, CI/CD, документация.

## Как запустить

```bash
# установить зависимости
pnpm install

# (опционально) поставить зависимости в подпроектах
cd client && pnpm install
cd ../server && pnpm install
cd ..

# запустить клиент и сервер параллельно
pnpm dev

# После запуска:

- Клиент: http://localhost:5173
- Сервер (health-check): http://localhost:3000/health
- WebSocket: ws://localhost:3000
